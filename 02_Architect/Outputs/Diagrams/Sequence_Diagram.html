🔄 Sequence Diagram — The Fat Milk

Luồng gọi API cho 3 use-case chính theo Clean Architecture

UC-1 Khách đặt hàng Online (PWA)
PWA Client
OrderController
OrderService
PostgreSQL
1PWA→ControllerPOST/api/orders { items, payment, voucher, delivery }
2Controller→ServiceValidate input (Zod), extract JWT user
3Service→DBBEGIN TRANSACTION
4Service→DBSELECT stock FROM inventory FOR UPDATE (lock rows)
5Service→DBCheck stock ≥ quantity → nếu không: ROLLBACK, ERR_OUT_OF_STOCK
6Service→DBValidate voucher (hạn, hạng, chi nhánh, usage_limit)
7Service→DBTính final_total = subtotal - tier_discount - voucher - points
8Service→DBNếu payment=wallet: SELECT balance FROM wallets FOR UPDATE
9Service→DBUPDATE wallet SET balance = balance - total (check ≥ 0)
10Service→DBUPDATE inventory SET stock = stock - quantity
11Service→DBINSERT orders + order_items + wallet_tx + points_ledger
12Service→DBCOMMIT
13Service→ServiceEvent: NotificationService.send(order_created)
14Controller←ServiceReturn order object
15PWA←Controller200 OK { order: { id, number, total, status } }
UC-2 Nạp ví qua SePay Webhook
SePay
WebhookController
WalletService
PostgreSQL
1SePay→ControllerPOST/api/webhook/sepay { id, amount, content, signature }
2Controller→ControllerVerify HMAC signature → nếu sai: 400 ERR_SEPAY_SIGNATURE
3Controller→ServiceParse content "TFM NAPVI 0901234567" → extract phone
4Service→DBCheck idempotency: sepay_transaction_id đã tồn tại? → skip
5Service→DBBEGIN TRANSACTION
6Service→DBSELECT balance FROM wallets WHERE user.phone = X FOR UPDATE
7Service→DBUPDATE wallet SET balance = balance + amount
8Service→DBINSERT wallet_transactions + sepay_transactions
9Service→DBCOMMIT
10Service→ServiceNotificationService.send(wallet_topup)
11Controller←Service200 OK (bắt buộc trả 200 để SePay không retry)
